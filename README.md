# Energy Load Forecasting API (XGBoost + FastAPI)
## Overview
This project implements a production-ready machine learning inference service for short-term energy load forecasting.
The system predicts the next 24 hours of electricity load using an optimized XGBoost regression model and exposes the prediction via a REST API built with FastAPI.

The objective is to simulate a real-world ML deployment scenario where a trained time-series model is served through an API for downstream integration.

## Problem Statement
Accurate short-term load forecasting is critical for:
- Energy demand planning
- Grid stability
- Operational cost optimization
- Peak load management
This project focuses on multi-step forecasting (24-hour horizon) using supervised learning with engineered time-series features.

## Model Architecture
- Model: XGBoost Regressor
- Forecast Horizon: 24 hours ahead
- Strategy: Multi-output regression

Evaluation Metrics:
- MAE
- RMSE
- MAPE
Baseline comparison performed against naive forecasting.

### Baseline comparison performed against naive forecasting.

| Model     | MAE    | RMSE   | MAPE  |
|:----------|-------:|-------:|------:|
| Baseline  | 161.77 | 215.75 | 8.10% |
| XGBoost   | 20.91  | 29.96  | 1.03% |

## API Architecture
- Framework: FastAPI
- Inference Engine: XGBoost (serialized via joblib)
- Deployment Ready: Yes (ASGI compatible)

POST `/predict`

## How to Run Locally
Create a virtual environment
```
python -m venv venv
venv\Scripts\activate  # Windows
```

Install dependencies
```
pip install -r requirements.txt
```

Run API
```
uvicorn app:app --reload
```

Access Documentation
```
http://127.0.0.1:8000/docs
```

## Technical Highlights
- Multi-output forecasting approach
- Walk-forward validation implemented during experimentation
- Residual diagnostics performed to validate randomness assumption
- Model evaluation using multiple error metrics
- Clean API separation between inference layer and serving layer
- Environment isolation using virtual environment
